## 2.3 非功能需求（Non-Functional Requirements）

- [ ] NFR-01: 响应延迟 ≤ 3秒（前端消息收发）
- [ ] NFR-02: 支持并发查询（MVP 阶段：单机，5并发足够）
- [ ] NFR-03: API Key 安全存储（不明文硬编码）
- [ ] NFR-04: 轻量化部署（SQLite + FAISS，无需外部服务依赖）
- [ ] NFR-05: README 覆盖本地启动步骤

## 2.4 验收标准

- [ ] AC-01: Demo 中 Telegram Bot 可接收并正确回答来自 HR KB 的问题
- [ ] AC-02: 上传 1 份 PDF 和 1 份 DOCX，两者均能被检索
- [ ] AC-03: 定义3个默认意图空间，查询分类准确率 ≥ 70%
- [ ] AC-04: Admin UI 5个页面可访问，核心操作可用
- [ ] AC-05: 查询历史可查看（至少包含意图和时间戳）
