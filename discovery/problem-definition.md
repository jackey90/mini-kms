# 问题定义与价值验证 — IntelliKnow KMS

## 1. 问题陈述

### 核心问题

企业在知识管理上面临三个相互关联的核心问题：

**问题 1：信息碎片化**
企业知识分散在邮件、共享盘、Wiki 页面和聊天记录中，缺乏统一的结构化存储。员工找到相关文档的平均时间超过 10 分钟，且往往找到的是过期版本。

**问题 2：检索效率低**
现有知识库工具（如 Confluence）提供全局关键词搜索，但无法理解查询意图。当 HR 员工问"年假政策"时，搜索结果可能混入财务、法务相关文档，用户需要手动筛选。

**问题 3：沟通渠道孤岛化**
员工日常工作在 Teams / Telegram 中，但知识库在独立的 Web 界面。这种渠道切换增加了使用摩擦，导致员工更倾向于直接@同事提问，而非查阅知识库。

---

## 2. 价值主张

### 核心价值主张

> **让企业知识触手可及 — 员工在哪里工作，知识就在哪里**

IntelliKnow KMS 解决方案：

| 问题 | 解决方案 | 价值 |
|------|---------|------|
| 信息碎片化 | 统一上传入口（PDF/DOCX）+ AI 自动解析结构化 | 一个地方管理所有企业知识 |
| 检索效率低 | 意图分类路由 + 向量语义检索（RAG） | 精准答案，减少搜索噪音 |
| 渠道孤岛化 | Telegram Bot + Teams Bot 原生集成 | 无需离开工作界面 |

### 差异化价值

- **对 Admin**：10分钟内完成文档上传到可查询的全流程；Analytics 提供知识库使用洞察
- **对 End User**：自然语言提问，3秒内得到带来源引用的精准答案
- **对企业**：自托管部署，数据不外传；Docker Compose 一键启动，无运维复杂度

## 3. 成功度量标准

### MVP 验收指标（Go/No-Go）

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| Bot 响应时延 | ≤ 3 秒 | 端到端测试（从发送到收到回复） |
| 文档格式支持 | ≥ 2 种（PDF + DOCX） | 上传各格式文档后可被查询 |
| 意图分类准确率 | ≥ 70% | 手工验证 10 条测试查询 |
| 前端集成数 | ≥ 2（Telegram + Teams） | 两个 Bot 均可收发消息 |
| 管理界面可用性 | 5个页面全部可访问 | 功能测试通过 |

### 北极星指标（长期）

- **员工查询成功率**：Bot 回答"找到相关内容"的比例 ≥ 80%（长期目标）
- **管理员上传效率**：文档从上传到 Processed 状态 < 2 分钟
- **知识库覆盖率**：最高频的 Top 20 员工问题有对应知识条目

## 4. 假设清单与验证计划

### 核心假设

| ID | 假设 | 风险级别 | 验证方式 |
|----|------|---------|---------|
| A-01 | OpenAI API zero-shot 分类准确率 ≥ 70% | 高 | 用10条测试查询验证，MVP前必须通过 |
| A-02 | FAISS 向量检索在小型 KB（<100文档）下延迟 ≤ 1s | 中 | 性能测试 |
| A-03 | Telegram + Teams Bot 集成可在7天内完成 | 中 | Teams Bot Framework 注册流程评估 |
| A-04 | LangChain PDF/DOCX Loader 能处理带表格的文档 | 中 | 上传含表格的 HR 工资文档验证 |
| A-05 | Docker Compose 能在标准开发机上10分钟内启动 | 低 | 在干净环境测试 |

### 验证优先级

1. **立即验证（开发前）**：A-01 意图分类准确率 — 用 Jupyter Notebook 快速测试 OpenAI zero-shot 分类效果
2. **开发中验证**：A-03 Teams 集成复杂度 — Day 1 先评估 Bot Framework 注册流程
3. **开发完成后验证**：A-02、A-04、A-05 — 集成测试阶段验证

## 5. 场景优先级（PRD 输入）

基于用户研究和价值主张，功能优先级排序：

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0 | Telegram Bot 接收并回答查询 | 核心价值交付，验证 End User 体验 |
| P0 | PDF/DOCX 上传与 RAG 检索 | 无知识库则 Bot 无内容可答 |
| P0 | 意图分类路由（3个默认空间） | 差异化核心，准确率是验收门槛 |
| P1 | Admin Dashboard（KB + Intent 页面） | Admin 操作入口，影响文档管理效率 |
| P1 | Teams Bot 集成 | 第二个前端集成，面试要求 ≥ 2 |
| P2 | Analytics 页面 | 锦上添花，不影响核心流程 |
| P2 | 数据导出 | 功能完整性 |
